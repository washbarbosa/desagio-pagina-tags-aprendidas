Comunicação Client x Server

Client

O cliente é o agente do usuário, que pode ser um navegador da Web ou até mesmo um programa usado por engenheiros e desenvolvedores da Web para depurar suas aplicações. O navegador é sempre a entidade que inicia a solicitação. Para exibir uma página da Web, o navegador envia uma solicitação original para buscar o documento HTML que representa a página. Ele também faz solicitações adicionais para scripts de execução, informações de layout (CSS), imagens e vídeos contidos na página. O navegador combina esses recursos para apresentar o documento completo, ou seja, a página da Web.

Server

O servidor é o responsável por lidar com as solicitações do cliente e fornecer respostas. Quando o servidor recebe uma solicitação, ele processa e envia uma resposta. Essa resposta pode incluir o próprio documento HTML, imagens, scripts ou outros recursos solicitados. O servidor é a entidade que responde às mensagens do cliente.

Como funciona a comunicação ?

O usuário digita uma URL no navegador. Esse URL aponta para um recurso específico na Web, como uma página da Web ou um arquivo.

O navegador envia uma mensagem de solicitação HTTP ao servidor. Essa mensagem é compreensível tanto pelo cliente quanto pelo servidor. É como ir a uma loja e pedir algo dizendo "olá".

O servidor recebe a solicitação e procura o endereço do servidor da Web associado ao URL. Isso é feito por meio do DNS. O DNS traduz o nome do domínio (como "www.example.com") em um endereço IP.

O servidor responde com o endereço IP do servidor da Web.

O navegador estabelece uma conexão com o servidor da Web usando esse endereço IP.

O servidor processa a solicitação e envia uma resposta HTTP de volta ao navegador. Essa resposta pode incluir o conteúdo da página da Web solicitada ou outros recursos.

Estrutura Básica do HTML

O HTML é uma linguagem de marcação que define a estrutura de uma página da Web. Aqui está o esqueleto básico de um documento HTML:

A estrutura básica do HTML consiste em elementos essenciais para criar uma página web. Inclui o doctype, que define a versão do HTML, as tags "html" para envolver todo o conteúdo, "head" para informações do documento, como metadados e links para estilos e scripts, e "body" para o conteúdo visível na página. Essa estrutura fornece o esqueleto básico para construir uma página web e é o ponto de partida para adicionar conteúdo, estilos e funcionalidades.

Falando sobre Tags e seus Atributos: As tags HTML são elementos fundamentais para estruturar e formatar o conteúdo de uma página web. Elas são cercadas por colchetes angulares e podem ter atributos para definir características adicionais. Dentre elas temos:

  • "h1" a "h6": Para títulos de diferentes níveis de importância.
  • "p": Para parágrafos de texto.
  • "a": Para criar links para outras páginas ou recursos.
  • "ul" e "ol": Para listas não ordenadas e ordenadas, respectivamente.
  • "li": Para itens de lista.
  • "img": Para inserir imagens na página.

Textos: Além das tags citadas acima, é possível estilizar o texto usando atributos como styles e classes, e é possível formatar o texto usando tags como "br" para quebras de linha e "hr" para linhas horizontais. Essas tags ajudam a estruturar e apresentar o conteúdo textual de maneira organizada e legível em uma página web.

Listas Ordenadas e Não Ordenadas: São elementos HTML que permitem organizar e exibir informações de maneira estruturada em uma página web. Listas Não Ordenadas, são listas que não seguem uma ordem específica. Cada item é marcado com um ponto, um círculo ou um quadrado. São definidas usando a tag "ul" e cada item da lista é definido com a tag "li". Listas Ordenadas, são listas que seguem uma ordem numérica ou alfabética. Cada item é automaticamente numerado. São definidas usando a tag "ol" e cada item da lista é definido com a tag "li". Ambas as listas são úteis para organizar itens relacionados e são amplamente utilizadas em conteúdo web para melhorar a legibilidade e a estrutura do texto.

Links: Links no HTML são elementos usados para criar conexões entre diferentes páginas da web ou recursos online. Eles são definidos pela tag "a" e podem ser usados para linkar para páginas externas". Utilizando o atributo "href", você pode especificar o endereço URL de destino. Como também linkar para páginas internas. Os links são uma parte fundamental da experiência de navegação na web, permitindo aos usuários acessar e interagir com diferentes recursos e conteúdos online.

O Que Você Precisa Saber Sobre Git e GitHub?: O Git é um sistema de controle de versão distribuído usado para rastrear alterações no código fonte durante o desenvolvimento de software. Ele permite que você mantenha um histórico de todas as alterações feitas em seus arquivos, o que facilita o trabalho em equipe, o controle de versão e a reversão para versões anteriores do seu projeto. Já, o GitHub é uma plataforma de hospedagem de código fonte baseada na web que utiliza o Git para controle de versão. Ele permite que você hospede seus repositórios Git na nuvem, facilitando o compartilhamento de código com outras pessoas e colaboração em projetos. Além disso, o GitHub oferece recursos adicionais, como rastreamento de problemas, solicitações de pull e integração contínua, que podem melhorar o fluxo de trabalho de desenvolvimento de software. Ao começar no desenvolvimento web com HTML, você pode se beneficiar do Git e do GitHub de várias maneiras, como usar o Git para controlar as alterações no seu projeto HTML, permitindo que você acompanhe o progresso do desenvolvimento e reverta para versões anteriores se necessário. Com o GitHub, você pode compartilhar seu código com outras pessoas, permitindo colaboração em projetos HTML, revisão de código e feedback de colegas. E Ao hospedar seu projeto HTML no GitHub, você pode criar um portfólio online de seus trabalhos, o que pode ser útil para mostrar suas habilidades e experiência a potenciais empregadores ou colaboradores. Começar a usar Git e GitHub pode parecer um pouco intimidante no início, mas eles são ferramentas poderosas que podem melhorar significativamente seu fluxo de trabalho de desenvolvimento e colaboração. Existem muitos recursos online, tutoriais e cursos disponíveis para ajudá-lo a aprender e dominar essas ferramentas ao longo do tempo.
